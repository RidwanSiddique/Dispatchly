Cross-Cutting Concerns

🔔 Notification Engine
Every significant action (ticket created, status changed, assigned, escalated, SLA alert, approval needed) triggers an INSERT into notifications table. The frontend bell polls and marks read.
⏱ SLA Engine
SLA minutes are set per priority (P1=60m, P2=240m, P3=480m, P4=4320m). computeSla() calculates % elapsed. Cron checks every 5min — fires alerts and auto-escalates on breach.
🛡 Role-Based Access
7 roles: admin, manager, agent, technician, specialist, hr, client. REQUESTER_ROLES (hr, client) see only their own tickets. Technicians see only assigned tickets. Admin bypasses all role checks.

Roles & Access Matrix

7 roles, each with different visibility and action permissions

admin
Full access to everything. Bypasses all role checks. Manages users, schedules, approvals. Sees all tickets.
manager
Sees all tickets. Can assign, escalate, resolve. Approves changes. Manages on-call. Gets SLA + new ticket notifications.
agent
Sees all tickets. Can create, comment, escalate, resolve. Can convert tickets to KB articles.
technician
Sees only tickets assigned to them. Must accept/decline assignments. Can resolve. Gets SLA alerts for assigned tickets.
specialist
Sees all tickets. Can manage KB articles (create, edit, delete). Can resolve tickets.
hr
Requester role. Can only see tickets they created. Priority fixed to P3. Cannot post internal comments.
client
Requester role. Same as HR. Can only see tickets they created. Priority fixed to P3.
